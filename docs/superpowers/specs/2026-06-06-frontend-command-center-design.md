# 竞品同步工具前端重设计说明

## 背景

当前前端已经具备完整功能，但首页的信息重心过于平均，呈现方式更像“配置页 + 结果页”的叠加，进入应用后缺少清晰的首要动作。对于高频执行同步任务的桌面工具，这会拉长确认路径，也削弱“打开即开跑”的效率感。

本次重设计目标不是改变业务流程，而是在保留现有功能与信息架构的前提下，重做界面层级、视觉语言与交互节奏，把应用调整为更接近“冷静专业的深色指挥台”。

## 已确认方向

- 改造力度：大幅重设视觉语言
- 首屏核心任务：立即开始同步
- 产品气质：指挥台型桌面工具
- 情绪基调：冷静专业
- 首页方向：A，发射台式首页

## 设计目标

1. 首页进入后，用户应在 3 秒内理解当前是否可以执行同步，以及本次会执行什么。
2. “开始同步任务”必须成为首页唯一主动作，其他信息围绕该动作服务。
3. 日志、异常留痕、快速入口必须保留，但视觉优先级低于执行动作。
4. 分组编辑和抓取设置继续存在，但退出首页主舞台，收纳到更安静的配置区域。
5. 整体界面应摆脱通用深色后台观感，更像专业桌面工具而非普通业务后台。

## 信息架构

### 首页

首页重构为任务发射台，只承接两类需求：

- 执行前确认：站点、模式、已选分组数量、是否存在阻断信息
- 执行中观察：关键日志、待执行分组、快速动作、异常提醒

首页不再承担完整配置编辑职责。配置相关深操作继续保留，但移入第二个标签页。

### 配置页

配置页继续保留双栏结构，但职责更清晰：

- 左侧为分组编辑区，负责当前分组选择、命名、ASIN 录入与输出路径预览
- 右侧为抓取设置区，负责站点、浏览器模式、邮编及等待时间等环境参数

配置页只保留一个高权重提交动作，即“保存设置”。“恢复默认”保留，但不与保存按钮争夺视觉主导权。

## 首页结构设计

### 1. 主操作区

主操作区位于首页首屏核心位置，是视觉与交互的中心。该区域包含以下内容：

- 一行运行摘要，展示站点、浏览器模式、已选分组数、最近完成时间
- 主标题，直接表达当前任务状态，例如“已准备同步 3 个分组”
- 唯一主按钮，文案为“开始同步任务”
- 轻量状态卡，仅保留执行决策相关摘要，不扩展为传统数据看板
- 阻断提示区，用于展示无可执行分组、配置保存失败等必须先处理的问题

该区域的目标是让用户在极短时间内做出“现在就执行”或“先调整”的判断。

### 2. 运行轨迹区

日志区从“完整堆叠的文本输出”调整为“关键事件流”。默认展示最近几条高价值事件，例如：

- 某个分组开始执行
- 某个 ASIN 抓取完成
- 任务结束
- 重试或失败信息

用户仍然可以滚动查看完整日志，但默认视觉表达更强调节奏与进度，而不是把整块页面做成终端输出。

### 3. 任务侧栏

首页右侧设置固定侧栏，承接与主动作强相关但不应抢主视觉的信息：

- 待执行分组：仅展示本次被选中的分组及其 ASIN 数量
- 快速动作：打开今日 Excel、打开 data 目录、打开日志目录、进入高级设置
- 异常提醒：提示失败截图、价格调试 JSON、站点相关限制等

这部分信息应该始终可用，但必须保持安静和可信，不制造视觉噪声。

## 关键组件设计

### 主标题与状态文案

首页的大标题不再强调品牌名，而强调当前任务状态。品牌名退到应用顶栏或辅助位置。标题文案以动作语义为主，例如：

- 已准备同步 3 个分组
- 当前没有可执行分组
- 同步任务执行中

配套说明文本保持简短，只解释当前状态，不重复页面名称。

### 状态卡

状态卡只保留与本次执行直接相关的信息，例如：

- 站点
- 浏览器模式
- 待执行分组数
- 最近一次完成时间

不扩展到与当前决策无关的泛指标，避免落入常见 SaaS 数据卡模板。

### 按钮体系

- 首页只保留一个高权重主按钮：“开始同步任务”
- 快速入口按钮统一为次级按钮
- 配置页只保留一个高权重提交按钮：“保存设置”
- 危险动作如删除分组继续保留明显区分，但不做戏剧化强化

### 日志项

日志项以列表形式展示，单条信息强调动作和对象，不做冗长说明。需要保留时间、阶段、结果三个要素，便于扫读和回溯。

## 视觉语言

### 配色策略

整体采用冷感深色体系，避免当前偏暖金的装饰倾向。建议采用以下角色分工：

- 背景色：冷黑蓝，作为桌面工具整体底色
- 主操作区：比背景更亮一层，用于聚焦任务启动
- 侧栏与日志区：较主操作区略收，形成稳定层级
- 主强调色：钢蓝或冰蓝，仅用于主按钮、聚焦边框、关键状态和少量高亮文本
- 危险/错误色：保持谨慎使用，仅用于异常或删除类动作

避免把大量半透明磨砂卡片叠满页面，也避免暖色大面积出现导致气质偏离“冷静专业”。

### 形状与边框

- 容器圆角控制在专业工具的稳定区间，避免过度圆润
- 不使用“浅边框 + 大范围软阴影”的通用卡片模式
- 通过明暗层级、局部描边和密度差构建结构，不依赖装饰性特效

### 排版

- 中文界面以简短、直接、任务导向的文案为主
- 模块标题使用操作语义，如“待执行分组”“运行轨迹”“快速入口”“异常留痕”
- 通过字号、字重、留白建立明确层级，避免目前大面积相近灰度文本造成的层级模糊
- 品牌名称弱化，任务状态强化

## 动效原则

动效只用于增强状态感知，不做装饰：

- 主按钮提供明确悬停、按下和禁用反馈
- 标签切换使用短距离位移或交叉淡入
- 新日志追加时使用轻量淡入
- 同步执行中，主操作区提供持续但安静的状态反馈

所有动效都应提供低干扰体验，不使用跳动、弹性或明显炫技效果。

## 交互规则

1. 当没有可执行分组时，主按钮禁用，并在主操作区给出明确原因。
2. 当配置未保存但用户准备执行同步时，应优先保存配置，再进入执行流程。
3. 首页展示的是“本次会执行什么”，而不是“系统里总共有多少东西”。
4. 配置页修改分组或参数后，应给出明确但不吵闹的保存提示。
5. 错误信息优先表达“发生了什么”和“后续去哪看”，例如日志目录、截图目录、调试 JSON。

## 实现边界

本次重设计只调整前端界面层，不改变以下内容：

- IPC 接口契约
- 同步主流程
- 分组与配置的数据结构
- 日志和文件落盘机制
- Excel 输出逻辑

如果实现过程中发现现有状态组织妨碍界面重构，可以做最小必要的前端状态整理，但不扩展为无关重构。

## 实现与验证要求

实现阶段需要同时验证以下状态，避免只完成静态重排：

- `tracker` 不可用时的 preload 错误提示
- 没有可执行分组时的首页禁用态
- 同步进行中的按钮、日志和摘要状态变化
- 配置修改后未保存、保存成功、保存失败三类提示
- 美区与欧洲站点切换后的设置显隐与说明文案
- 窄窗口下首页主操作区、侧栏和日志区的层级稳定性

验证优先级以高频路径为先，即“打开应用 -> 确认分组 -> 开始同步 -> 观察进度 -> 打开结果或日志”。

## 验收标准

1. 首页首屏能明确突出“开始同步任务”。
2. 用户不进入设置页也能完成高频同步路径。
3. 关键日志、待执行分组、快速入口和异常留痕在首页可见。
4. 设置页仍完整保留分组编辑与抓取参数配置能力。
5. 新界面在桌面窗口下保持稳定层级与良好可读性，不出现文本拥挤、动作冲突或视觉平均分配问题。
